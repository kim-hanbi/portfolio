네트워크 보안 실습 환경 구축 (Snort)

Firewall · IDS/IPS · HIDS · VPN 통합 구성 및 탐지 규칙 작성

본 프로젝트는 네트워크 보안 환경을 직접 설계·구축하고 다양한 공격 탐지 규칙을 적용해보는 실습입니다. 방화벽, IDS/IPS, HIDS, VPN을 통합적으로 구성하여 공격을 탐지하고 대응하며, 이를 통해 실무에 가까운 보안 운영 경험을 쌓는 것을 목표로 하였습니다.

과정: Firefox3 VM에서 HTTP 요청 및 Telnet 연결 시도

결과: HTTP 허용 / Telnet 차단

의미: 서비스별 접근 제어가 정상적으로 동작함을 확인

2. Webterm2 → DMZ

목표: DMZ 영역 접근 시 Telnet 통신 검증

결과: Telnet 허용

의미: DMZ 구간 접근 정책이 정상적으로 동작함을 확인

HSRP 결과

목표: 라우터冗長성(이중화) 확인

결과: R3 라우터가 Standby, Active 라우터 정상 동작

의미: 장애 발생 시 자동으로 역할 전환이 가능함을 확인

OpenVPN 실습

목표: 원격 접속 환경에서 안전한 내부망 접근

결과: Windows10 클라이언트에서 VPN 접속 성공, 내부 네트워크 접근 확인

의미: 원격 환경에서도 안전한 네트워크 접근이 가능함

Snort Rule 설정 및 탐지

다양한 공격 유형 탐지를 위해 Snort 규칙을 작성 및 적용하였습니다.

예시 Snort Rule

alert icmp any any -> 192.168.1.0/24 any (msg:"ICMP Ping Detected"; sid:1000001; rev:1;)

이 규칙은 내부 네트워크로 향하는 Ping 요청(ICMP)을 탐지합니다.

Kali → CentOS7 (SYN Scan 탐지)

결과: Snort 로그에 SYN Scan 탐지

의미: 포트 스캐닝 활동을 조기 식별 가능

결과 요약

  • Firewall: 서비스별 접근 제어 정책 검증 완료
  • HSRP: 라우터 이중화 성공
  • VPN: 안전한 원격 접속 환경 구축
  • Snort/HIDS: 다양한 공격 시나리오 탐지 성공

개선점 및 배운 점

  • Snort 규칙 민감도 조정 과정에서 오탐 발생 → 규칙 튜닝 경험
  • VPN 환경은 장시간 부하 테스트 부족 → 추후 보완 필요
  • 보안 운영에서 단순 탐지뿐 아니라 **로그 분석과 대응 절차**의 중요성 인식
